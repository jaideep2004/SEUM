\# SEUM UI \& UX Standards

Version: 2.0



\## Design Vision



SEUM is a premium enterprise Transportation Operating System.



The interface should feel like a combination of



\- Stripe Dashboard

\- Linear

\- Shopify Admin

\- Vercel

\- Modern Aviation Control Systems



NOT



\- Bootstrap Admin

\- Material Dashboard Templates

\- Government ERP

\- Old Enterprise Software



The UI should communicate



Professionalism



Operational Control



Trust



Speed



Clarity



\---



\# Core Principles



Priority



1\. Readability

2\. Consistency

3\. Visual Hierarchy

4\. Information Density

5\. Fast Navigation

6\. Accessibility



Every screen should reduce cognitive load.



\---



\# Overall Layout



Application layout always consists of



Dark Left Sidebar



Top Header



Page Header



KPI Section



Main Content



Supporting Panels



Footer (optional)



Never redesign the layout between modules.



\---



\# Sidebar



The sidebar is part of the SEUM identity.



Always



Dark navy gradient



Company logo



Company switcher



Grouped navigation



Rounded active item



Bottom user profile



Support section



Do not use floating navigation.



\---



\# Content Area



Content area should remain



Bright



Clean



Spacious



Professional



Large margins



Comfortable spacing



Never make pages feel cramped.



\---



\# Cards



Cards are the foundation of the interface.



Default



Radius



16px



Padding



24px



Border



1px



Soft shadow



White background



Every card contains



Title



Optional subtitle



Primary content



Optional actions



Never overcrowd cards.



\---



\# KPI Cards



Large icon



Primary metric



Supporting text



Trend indicator



Optional sparkline



Use identical sizing across dashboards.

\---



\# Color System



Primary



Blue



Success



Green



Warning



Amber



Danger



Red



Info



Purple or Blue



Neutral



Gray



Never invent additional status colors.



Use color to communicate state, not decoration.



\---



\# Charts



Charts should be



Minimal



Readable



Lightweight



High contrast



No heavy gradients



No 3D effects



Support



Line



Area



Bar



Donut



Progress



Gauge



\---



\# Tables



Tables are primary business components.



Must support



Search



Sorting



Filtering



Pagination



Status badges



Row actions



Sticky headers



Comfortable row height



Never compress tables excessively.



\---



\# Maps



Maps are first-class components.



Support



Live GPS



Markers



Routes



Status colors



Popup cards



Fullscreen



Filtering



Maps should occupy significant visual space when present.



\---



\# Status Badges



Use soft backgrounds.



Rounded pill badges.



Examples



Active



Online



Delayed



Maintenance



Offline



Completed



Cancelled



Keep badge styling consistent.



\---



\# Forms



Split into logical sections.



Group related fields.



Show helper text.



Inline validation.



Avoid long uninterrupted forms.



\---



\# Icons



Use Lucide icons only.



Consistent stroke width.



20–24px default.



Icons should improve scanning, not decorate.

\# Visual Style



Use



Soft shadows



Thin borders



Rounded corners



Large whitespace



Subtle separators



Avoid

Glassmorphism

Overly decorative effects



\---



\# Responsive



Desktop First



Tablet Supported



Mobile Functional



Business actions must remain accessible.



\---



\# Accessibility



Readable contrast.



Visible keyboard focus.



Minimum body font



15px



Touch targets



Minimum



40px



Never sacrifice usability for aesthetics.



\---



\# Consistency Rules



Every dashboard must share the same



Sidebar



Header



Card styles



Spacing



Typography



Buttons



Tables



Charts



Badges



Users should instantly recognize every page as SEUM.



\---



\# Golden Rule



Design for operators who use the system 8+ hours every day.



Optimize for speed, clarity and consistency before visual creativity.
